# L02 Transformer 架构核心原理

> **一句话精髓**：**注意力机制 = 阅读时知道该重点看哪个词**——模型对每一个位置都算「我和谁更相关」，相关就多「看」几眼，把信息汇总结论写进新的表示里。

## 一、从 RNN / LSTM 讲起：为什么它们「被淘汰」？

### 1.1 RNN 在做什么？

循环神经网络（RNN）按**时间步**展开，每一步读一个词，带着「隐藏状态」往前走：

```
  x1 -> [h1] -> x2 -> [h2] -> x3 -> [h3] -> ...
         ^              ^              ^
         |______________|______________|
              信息通过隐藏状态传递
```

**类比**：你背课文时**只能记住一个「当前小结」**，每读一个新词，就把小结更新一下——越远的前文，越容易被「挤掉」或稀释。

### 1.2 LSTM / GRU 改进什么？

长短期记忆（LSTM）用**门控**缓解梯度消失，让模型更容易**选择性记住**长期信息。

```
        遗忘门 / 输入门 / 输出门
                |
                v
        细胞状态像「传送带」
```

但本质仍是**逐步串行**：算完第 t 步才好认真算第 t+1 步。

### 1.3 为什么在大模型时代不香了？

| 痛点 | 解释 |
|------|------|
| **并行差** | 句子一长，时间步串行，GPU 难以「一口气算完全句」 |
| **长依赖仍吃力** | 距离远了，信息仍可能衰减（LSTM 缓解但不根治） |
| **规模化效率** | 超大模型 + 超长上下文，RNN 架构扩展性吃亏 |

**面试金句**：RNN 类是**顺序依赖**；Transformer 用注意力把**依赖关系**显式建模，更利于**并行**与**长程**。

## 二、Transformer 的革命：Attention Is All You Need

### 2.1 2017 年那篇论文标题在说什么？

原论文：[Attention Is All You Need](https://arxiv.org/abs/1706.03762)（Google，2017）。

```
  「All You Need」有点宣传意味，但核心思想成立：
    用注意力机制替代/弱化传统 RNN 的递归依赖，
    让序列建模更并行、更可扩展。
```

### 2.2 宏观结构（先背这张图）

```
        输入 token 嵌入 + 位置编码
                    |
        +-----------v-----------+
        |   Encoder（若干层）    |   （机器翻译原文任务里用）
        +-----------+-----------+
                    |
        +-----------v-----------+
        |   Decoder（若干层）    |
        +-----------+-----------+
                    |
                 输出概率
```

**GPT 系列**主要用 **Decoder-Only**（只有右边decoder栈，且改成自回归生成）；**BERT**用 **Encoder-Only**；**原始翻译模型**是 **Encoder-Decoder**。

## 三、自注意力（Self-Attention）直觉：生活中的例子

### 3.1 读句子时的「眼神分配」

句子：「小明把苹果放在桌子上，它滚到地上了。」

```
  问：这里的「它」指什么？
  人脑会「多看几眼」苹果，而不是桌子。
```

Self-Attention 在做类似的事：对每个词位置，计算**和其他词的相关度**，相关度高就**多吸收**对方的信息。

### 3.2 用「开会」类比

每个词是一个**参会者**：

```
  轮到「它」发言总结时：
    · 问全场：谁和我最相关？（算注意力权重）
    · 按权重把别人的意见加权平均（信息聚合）
    · 形成「它」的新理解向量
```

### 3.3 与「卷积」对比（面试加分）

- **卷积**：看局部窗口，堆深层扩大感受野。  
- **Self-Attention**：**一步**让任意两位置可交互（全局），代价是计算与显存随长度上升（故有 FlashAttention 等优化）。

## 四、Q、K、V 是什么？——图书馆查书比喻

### 4.1 比喻设定

想象图书馆：

```
  · 你脑子里有一个 **问题（Query）**
  · 每本书封面上有 **标题关键词（Key）**
  · 书里有 **正文内容（Value）**
```

流程：

```
  1) 用你的 Q 和所有书的 K 做匹配，得到「哪几本书最相关」的分数
  2) 把分数归一化成权重（softmax）
  3) 按权重去混合那些书的 V，得到「针对这个问题的综合摘录」
```

### 4.2 在 Transformer 里

每个词位置都会长出三个向量（由输入嵌入线性变换得到）：

| 符号 | 名字 | 直觉 |
|------|------|------|
| **Q** | Query | 「我在问什么」 |
| **K** | Key | 「我提供什么索引/标签便于被匹配」 |
| **V** | Value | 「我真正携带的信息内容」 |

**注意**：Q/K/V 都是**学出来的**，不是人工规则；同一句话里每个 token 都有自己的 Q、K、V。

### 4.3 形状心智（背下来）

设 batch 先忽略，头数先为 1：

```
  序列长度 L，维度 d_model

  X  (L × d_model)
  Wq, Wk, Wv  (d_model × d_k 或 d_model × d_v)

  Q = X Wq   -> L × d_k
  K = X Wk   -> L × d_k
  V = X Wv   -> L × d_v
```

**Scaled Dot-Product Attention**：

```
  Scores = Q K^T / sqrt(d_k)     -> L × L
  Weights = softmax(Scores)      -> 每行对列归一化
  Output = Weights V             -> L × d_v
```

**sqrt(d_k)** 的作用：点积方差随维度变大，softmax 会**更尖**（梯度不稳）；缩放使训练更稳。

## 五、多头注意力（Multi-Head Attention）

### 5.1 为什么一头不够？

**类比**：你读论文时，有人关注「方法」，有人关注「实验」，有人关注「相关工作」——**不同子空间关注不同模式**。

```
  Head 1: 句法结构
  Head 2: 指代消解
  Head 3: 语义搭配
  ...
```

### 5.2 计算上怎么做？

```
  对每个 head i：
      Qi, Ki, Vi = 各自的线性投影
      head_i = Attention(Qi, Ki, Vi)

  Concat(head_1, ..., head_h) -> 再线性投影融合
```

**面试考点**：多头**不是**简单重复，而是**不同投影矩阵**学到不同子空间；最后 **concat + Wo** 再混合。

## 六、位置编码（Positional Encoding）

### 6.1 问题：Self-Attention 本身是「置换不变」的

如果只喂词嵌入，句子「猫吃鱼」和「鱼吃猫」可能被当成**同一袋词**（顺序信息丢光）。

```
  解决：给每个位置加一个位置向量，加到输入上
```

### 6.2 原版正弦余弦编码（了解即可）

不同维度用不同频率的 sin/cos，使模型能**外推**到训练时未见过的长度（理论上）；实际现代模型常用 **可学习位置嵌入** 或 **RoPE** 等变体（LLaMA/Qwen 系常见）。

### 6.3 面试一句话

> **位置编码让 Transformer 知道 token 的顺序信息**；具体实现因模型家族而异。

---

## 七、Encoder-Decoder vs Decoder-Only

### 7.1 Encoder

- 可看**整句输入**（双向上下文）。  
- 适合「理解」任务，如 BERT 做分类、NER。

```
  [我 喜欢 深度 学习]
     双向互相看
```

### 7.2 Decoder（自回归）

生成第 t 个词时，只能看 **t 之前**（因果掩码），防止「偷看未来」。

```
  预测「学习」时只能看到「我喜欢深度」
```

### 7.3 Encoder-Decoder

经典机器翻译：**编码器**读源语言，**解码器**生成目标语言，解码器还通过 **cross-attention** 看编码器输出。

```
  Source -> Encoder -> 记忆表示
                           |
  Target -> Decoder -------+-> 生成
```

---

## 八、GPT 为什么选 Decoder-Only？

### 8.1 生成式预训练的自然选择

GPT 的训练目标是：**给定前缀，预测下一个 token**。这与 **因果 Decoder** 完全一致。

```
  「我喜欢」 -> 预测「吃」
  「我喜欢吃」 -> 预测「苹果」
```

### 8.2 与 BERT 的分工（常考对比）

| | BERT（Encoder） | GPT（Decoder） |
|--|-----------------|----------------|
| 上下文 | 双向 | 单向（因果） |
| 典型预训练 | MLM 掩码 | 下一词预测 |
| 天生生成 | 需额外头 | 直接续写 |

### 8.3 工程与生态

Decoder-only 架构在大规模自回归训练上**路径清晰**，开源生态（LLaMA、Qwen 等）高度一致，**MedicalGPT** 也主要围绕这类模型。

## 九、前馈网络（Feed Forward Network, FFN）

### 9.1 在每个 Transformer 层里

典型结构：

```
  子层1: Multi-Head Self-Attention + 残差 + Norm
  子层2: FFN（对每个位置独立） + 残差 + Norm
```

**FFN** 常是：

```
  Linear(d_model -> d_ff) + 激活（GELU/ReLU） + Linear(d_ff -> d_model)
```

**类比**：Attention 负责「**和谁聊**」，FFN 负责「**聊完以后自己消化、记忆、重组**」——**逐位置**的非线性变换，容量很大。

### 9.2 面试点

- FFN 通常 **d_ff ≈ 4 × d_model** 量级（视架构而定）。  
- **MoE**（混合专家）是把 FFN 换成「多专家路由」，省计算或增容量（进阶）。

## 十、Layer Normalization

### 10.1 做什么？

对**每个样本、每一层**的特征做归一化，减均值除方差，再仿射变换，**稳定训练**。

```
  类比：把一群人身高先标准化，再允许模型学「缩放与平移」
```

### 10.2 Post-Norm vs Pre-Norm（了解）

原版 Transformer 多为 **Post-LN**（残差后 Norm）；很多大模型用 **Pre-LN** 或 **RMSNorm** 以提升训练稳定性。面试可说：**Normalization 位置因模型而异**。

## 十一、从 Transformer 到 GPT 到 BERT：架构选型

```
                Transformer 论文
                        |
          +-------------+-------------+
          |                           |
    Encoder-Only                  Decoder-Only
    （BERT 系）                    （GPT 系）
          |                           |
    理解任务强                    生成任务强
    需加生成头                    直接续写
```

**T5、BART** 等是 Encoder-Decoder 路线，适合「输入一段输出一段」的转换任务；**聊天大模型**主流是 Decoder-Only + 对齐。

## 十三、面试高频题：解释 Self-Attention 计算过程

### 13.1 标准口述版（2 分钟）

1. **输入**：一串 token 嵌入，形状约 `L × d`。  
2. **线性投影**得到 Q、K、V。  
3. **相似度**：`QK^T`，除以 `sqrt(d_k)` 缩放。  
4. **因果掩码**（Decoder）：把未来位置的 logits 设为 **−∞**，softmax 后权重为 0。  
5. **softmax** 得到注意力权重，每行和为 1。  
6. **加权求和**：权重乘 V，得到输出；多头则并行多组再拼接融合。  
7. **残差与 Norm**：缓解梯度问题、稳定分布。  
8. **FFN**：逐位置非线性变换，增加表达力。

### 13.2 复杂度意识（加分）

- 注意力矩阵 `L × L`，显存与算力随序列长度**平方级**增长——故长上下文要 **FlashAttention、稀疏注意力、线性注意力** 等工程优化。

### 13.3 常见追问

- **为什么叫 self？** Q、K、V 都来自**同一序列**。  
- **cross-attention？** Q 来自解码器，K/V 来自编码器（翻译场景）。  
- **和 RNN 记忆区别？** RNN 压缩成固定隐藏状态；Attention 是**显式权重**从所有位置读信息。

## 附录 D：易错点澄清

**错：「注意力权重就是概率所以可解释」**  
正：权重是**模型内部相关性**，不等于人类因果；可解释性要谨慎。

**错：「Transformer 完全不要 RNN 所以没有顺序」**  
正：顺序靠**位置编码**；没有位置信息就会乱序。

**错：「QKV 是三种不同的词向量」**  
正：它们是**同一层对同一输入的三种线性投影**，语义是「角色」不是「三个词」。
