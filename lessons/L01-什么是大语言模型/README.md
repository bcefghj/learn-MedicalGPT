## 一、什么是人工智能（AI）？——用生活例子讲清楚

### 1.1 直觉类比：从「自动门」到「会聊天的程序」

想象三种东西：

```
  ① 普通门：你推才开
        |
        v
  ② 自动门：传感器检测到人 → 电机开门（规则很固定）
        |
        v
  ③ 「智能」助手：你说一句话，它尝试给出合理回应（规则极其复杂，常由数据学出来）
```

**人工智能（Artificial Intelligence，AI）**在通俗意义上，就是让机器表现出**需要智能才能完成**的行为：识别图像、听懂语音、下棋、推荐商品、**生成文字**等。

注意：日常说的「智能」常带有「像人一样理解」的味道，但工程上的 AI 更多是：**在大量数据与算力下，学会一个从输入到输出的复杂函数**。

### 1.2 强人工智能 vs 弱人工智能（面试常考概念）

| 说法 | 含义 | 类比 |
|------|------|------|
| **弱 AI / 窄 AI** | 只擅长某一类任务 | 会下围棋的程序未必会做饭 |
| **强 AI** | 通用、像人一样各方面都行 | 科幻里常见的「真正意识」 |

**今天的大语言模型**在对话、写作、代码上很强，但仍是**窄任务上的统计专家**，不要把它等同于「有自我意识的强 AI」——这是面试与科普里都要强调的边界。

### 1.3 本课里 AI 的「最小定义」（背这句就够）

> **AI = 让机器从经验（数据）中获得能力，从而在未知输入上做出有用预测或决策。**

后面讲的机器学习、深度学习，都是实现 AI 的**不同工具箱层级**。

## 二、什么是机器学习（Machine Learning）？

### 2.1 核心思想：别手写规则，让机器从例子里学

传统编程有时是：

```
  if 邮件含「中奖」「汇款」:
      标为垃圾邮件
```

但真实世界太复杂，规则写不完。**机器学习**换成：

```
  给机器很多「邮件 + 是否垃圾」的例子
        |
        v
  机器自己调整内部参数，使在训练例子上错误尽量少
        |
        v
  希望在新邮件上也能猜得准
```

**类比**：你不是教孩子背下「一万条交通规则」，而是给他看很多「路况 → 该怎么开」的案例，让他**归纳**出规律（尽管机器用的是数学，不是人类那种直觉）。

### 2.2 三种经典学习方式（建立心智模型）

```
        机器学习
            |
    +-------+-------+-------+
    |               |       |
  监督学习        无监督    强化学习
 (有标准答案)   (没标签)  (试错+奖励)
```

- **监督学习**：每条数据有「正确答案」，如「这张图是猫还是狗」。
- **无监督学习**：只有数据本身，让模型发现结构，如聚类、压缩表示。
- **强化学习**：智能体在环境里行动，根据奖励信号改进策略；**RLHF**（后面课程会讲）就是把「人类偏好」当成一种奖励信号。

**大语言模型的预训练**常被视为：**用海量文本做自监督**（用前文预测下文，标签来自文本自身），所以既有「监督」的味道，又不需要人类逐句标注。

### 2.3 面试一句话

> **机器学习是：用数据优化模型参数，使某损失函数下降，从而在新数据上泛化。**

## 三、什么是深度学习（Deep Learning）？

### 3.1 深度 = 很多层

**深度学习**是机器学习的子集，特指用**深层神经网络**（很多层非线性变换堆叠）来学习表示。

```
输入（如一句话的 token）
        |
        v
   [ 线性 + 非线性 ]  ← 第 1 层
        |
        v
   [ 线性 + 非线性 ]  ← 第 2 层
        |
       ...
        |
        v
   输出（如下一个词的概率分布）
```

**类比**：浅层模型像「只经过两道加工工序的工厂」；深度网络像「流水线很长」，每一层可以逐步从「笔画边缘」组合成「部件」再组合成「语义」——**层次化特征**。

### 3.2 为什么近年才爆发？

三要素（缺一不可）：

```
        大数据
           +
        强算力（GPU）
           +
      算法突破（如 Transformer）
           |
           v
      深度模型真正好用
```

### 3.3 面试区分题：机器学习 vs 深度学习

| 维度 | 机器学习（广义） | 深度学习 |
|------|------------------|----------|
| 特征 | 可手工设计特征 + 简单模型 | 常端到端，特征也学习 |
| 数据规模 | 小数据有时也能work | 大数据更吃香 |
| 可解释性 | 部分模型较好 | 常是「黑盒」（但有研究在改进） |

## 四、什么是自然语言处理（NLP）？

### 4.1 一句话

> **NLP = 让计算机处理、理解、生成人类语言**（文本、语音转文字后的内容等）。

### 4.2 任务举例（你会在医疗场景里见到）

```
  分词 / 词性标注 / 命名实体识别（NER，如识别药品名、疾病名）
        |
        v
  信息抽取（从病历抽结构化字段）
        |
        v
  分类（这条问诊是否紧急）
        |
        v
  生成（问诊答复、病历摘要、科普文章）
```

### 4.3 NLP 与大模型的关系

**传统 NLP**：往往「一个任务一个模型」，特征工程多。  
**大模型时代**：先训练一个**通用语言模型**，再**微调**或**提示**到具体任务——**统一接口**（说话就能用）是革命性变化。

## 五、什么是语言模型（Language Model）？

### 5.1 最朴素定义

> **语言模型（LM）**：给前面的词（或 token），预测**下一个词**的概率分布。

```
  输入：「我今天早上喝了一杯」
  输出：P(牛奶)、P(咖啡)、P(豆浆)、... 的概率
```

**类比**：手机输入法的「联想下一个词」就是极简语言模型；大模型是**超级放大版**，参数多、数据多、上下文长。

### 5.2 「像人」从哪里来？

模型并没有「真正知道」牛奶和咖啡的区别，它学到的是：

```
  在训练语料里，「喝了一杯」后面
  哪些词出现频率高、哪些搭配合理（统计 + 深层模式）
```

当规模足够大时，这些模式会**间接编码**很多知识——所以看起来像「懂很多」。

### 5.3 n-gram 与 神经网络语言模型（建立历史感）

| 类型 | 思路 | 局限 |
|------|------|------|
| **n-gram** | 数频率，只看前 n−1 个词 | 稀疏、记不住长依赖 |
| **神经语言模型** | 用网络压缩上下文 | 可学更平滑、更长程的模式 |

**面试考点**：语言建模的**训练目标**常是**最大似然**（让正确下一个词的概率变大）。

## 六、什么是大语言模型（LLM）？

### 6.1 「大」在哪里？

通常指：

```
  · 参数量巨大（十亿、百亿、千亿级）
  · 训练数据海量（互联网级语料 + 各类书籍代码等）
  · 训练算力巨大（大量 GPU/TPU、长时间）
  · 能力涌现（同一架构放大后出现更稳定的指令遵循、推理痕迹等）
```

**没有绝对数值门槛**，业界习惯把 **B（十亿）参数级以上**且具备通用对话/任务能力的模型叫 LLM——具体数字面试时可以说「行业惯例约 1B+，常见讨论 7B、13B、70B 等规格」。

### 6.2 LLM 与 ChatGPT 的关系

```
  基座 LLM（擅长续写）
        +
  指令微调（SFT：学会听人话）
        +
  对齐（RLHF / DPO 等：更安全、更有用）
        |
        v
  对话产品（如 ChatGPT）
```

本系列后面的 **MedicalGPT** 也是类似 pipeline，只是把**数据与场景**换成医疗。

### 6.3 类比总结（背给非技术同事听）

> **LLM = 读了非常多书的「概率续写引擎」 + 后期训练让它更像「听话的助手」。**  
> 它不是数据库精确查询，而是**生成**最像「合理人类会写的内容」。

## 七、GPT 系列发展史（GPT-1 → GPT-4 → ChatGPT）

### 7.1 时间轴心智图

```
  2018  GPT-1   证明 Transformer Decoder 做预训练可行
         |
  2019  GPT-2   规模增大，生成质量提升，「零样本」能力受关注
         |
  2020  GPT-3   「规模」+「提示」范式；Few-shot 惊艳
         |
  2022  ChatGPT  基于 GPT-3.5 一类模型 + 对齐，产品化爆发
         |
  2023+ GPT-4    多模态、更强推理与工具使用（公开细节较少）
```

### 7.2 每一代在「学什么」？

| 代际 | 关键词 | 小白理解 |
|------|--------|----------|
| **GPT-1** | 生成式预训练 Transformer | 先无监督预训练，再微调下游任务 |
| **GPT-2** | 更大、更通吃 | 同一个大模型，很多任务不用改结构也能试 |
| **GPT-3** | In-context learning | **在提示里给例子**，模型就能模仿风格做任务 |
| **ChatGPT** | 对话体验 + 安全对齐 | 不只是续写，而是**像助手一样答** |
| **GPT-4** | 能力综合提升 | 更强、更贵、闭源细节少；行业多作标杆参照 |

**面试提示**：能说清 **「预训练 → 指令微调 → 对齐」** 三阶段，就比只背「GPT 很厉害」强很多。

### 7.3 和本课路线的关系

你学 **MedicalGPT**，本质是在学：**开源社区如何用「类 ChatGPT 流水线」在医疗领域复现可控版本**——模型可能换成 LLaMA / Qwen 等，但**阶段划分**高度相似。

## 八、大模型的能力：文本生成、翻译、问答、代码……

### 8.1 能力地图（ASCII）

```
                    ┌──────────────┐
                    │   大语言模型   │
                    └──────┬───────┘
           ┌───────────────┼───────────────┐
           v               v               v
      ┌─────────┐     ┌─────────┐     ┌─────────┐
      │ 文本生成 │     │ 理解与抽取│     │ 推理与规划│
      │续写/摘要 │     │分类/NER │     │解题/代码 │
      └─────────┘     └─────────┘     └─────────┘
           |               |               |
           v               v               v
      翻译、改写      检索前结构化      Agent 工具调用
```

### 8.2 医疗场景里常见的「能力调用」

- **问诊辅助**：注意合规，通常是「辅助」而非替代医生。
- **病历摘要**：把长病程压缩成要点（仍需人工审核）。
- **科普**：把术语讲通俗（要防幻觉，后文讲）。
- **代码**：处理数据脚本、对接医院信息系统外的工具链等。

### 8.3 重要边界（能力 ≠ 正确）

模型会**流畅地**写，不代表**事实正确**。医疗场景必须把**安全、溯源、人工审核**放第一位。

## 九、大模型的局限性：幻觉、时效性、计算成本

### 9.1 幻觉（Hallucination）

**定义**：模型**一本正经地编造**不存在的事实、文献、剂量或指南条款。

```
  用户：这篇论文的结论是什么？
  模型：它说 XXX……（实际上论文不存在或结论相反）
```

**类比**：一个背题很熟的考生，**不会就说得像会**，而且语气特别自信。

**应对思路**（后续课程展开）：RAG（检索增强）、引用真实文献、微调领域数据、人在回路审核等。

### 9.2 时效性

训练数据有**截止时间**。新药、新指南、新疫情信息可能**不在模型权重里**。

```
  模型权重 = 「闭卷考试的大脑」
  检索/工具   = 「开卷考的参考书」
```

产品上要明确：**是否需要外接最新知识源**。

### 9.3 计算成本

```
  训练：一次性或周期性巨贵（卡时、电费、人力）
  推理：每次提问都要算很多矩阵乘法（要 GPU/量化/批处理优化）
```

**面试常问**：为什么企业在意「推理成本」？——因为用户量一上来，**token 计费**就是真金白银。

### 9.4 其他常见局限（补充）

- **偏见与安全问题**：训练语料里的偏见可能被放大。
- **长上下文仍有限**：再长也有窗口，不能当无限内存。
- **可解释性**：难以逐句证明「为什么这样答」。

## 十、为什么医疗领域需要专门的大模型？

### 10.1 通用大模型哪里「不够」？

```
  通用语料多        但医学「说法」更严谨、术语更细
        +
  用户问法随意      医疗需要更稳妥的拒答与转诊建议
        +
  法规与伦理        对幻觉容忍度极低
```

**不是**说通用模型完全不能用，而是：**在同样安全流程下，领域适配往往更省人工、更一致**。

### 10.2 「专门」通常指什么？

| 手段 | 作用 |
|------|------|
| **增量预训练（PT）** | 让模型熟悉医学文本分布 |
| **有监督微调（SFT）** | 学会按问诊/病历格式回答 |
| **对齐（DPO/RLHF）** | 更礼貌、更谨慎、更符合偏好 |
| **RAG** | 把答案拴在真实指南/病历上 |

**MedicalGPT** 就是把上述流水线**工程化**的开源参考实现——你学它，等于学**行业通用打法**。

### 10.3 类比（给完全外行解释）

> 通用大模型像**读过百科全书的通才**；医疗领域模型像**又在医院实习了很久的专科生**——**术语、格式、谨慎程度**更像真实工作场景。

## 附录 D：零基础常见疑问（对话体）

**问：AI 会不会很快取代医生？**  
答：当前技术路线下，更现实的是**辅助**（文书、检索、宣教草稿），最终诊断与处置责任仍在人与制度。你要在面试里体现的是：**合规、可追溯、幻觉防控**。

**问：学大模型要不要数学很好？**  
答：能懂**高中数学 + 基础概率**足够入门；想做研究再补线性代数与微积分。工程岗更看重**调参、数据、系统**。

**问：「参数」到底是什么？**  
答：可以先把参数理解成神经网络里**可调的数字旋钮**，训练就是让这些数字整体朝着「预测更准」的方向动。

**问：ChatGPT 是搜索引擎吗？**  
答：**不是**。搜索是检索已有网页；ChatGPT 主要是**生成**——除非外挂检索（RAG），否则不能当作实时百科。

**问：为什么说「统计」不丢人？**  
答：大模型的知识来自语料共现与深层模式，**不是魔法**。承认这一点，你才能正确设计评测与安全边界。

**问：医疗大模型数据从哪来？**  
答：公开数据、脱敏院内数据、合成数据等各有合规要求；**具体采集与脱敏**见后续 L12、L13。此处只需记住：**数据质量决定上限**。

**问：我只关心「调用 API」还要学这些吗？**  
答：若做应用集成，可略浅；若简历写「训练/微调 MedicalGPT」，**L01–L04 是必会地图**，否则面试官一问就穿帮。
